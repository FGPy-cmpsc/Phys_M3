# Моделирование магнитной пробки

Проект рассчитывает траекторию заряженной частицы в магнитном поле двух коаксиальных колец с током (магнитная ловушка).

## 1. Установка и использование

### Шаг 1. Клонировать проект

```bash
git clone https://github.com/FGPy-cmpsc/Phys_M3.git
cd Phys_M3
```

### Шаг 2. Создать и активировать виртуальное окружение

Windows PowerShell:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate
```

Linux/macOS:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### Шаг 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### Шаг 4. Запустить программу

Демонстрация (две частицы + ансамбль, результаты в `demo_output/`):

```bash
python3 demo.py
```

Расчёт одиночной траектории с параметрами по умолчанию:

```bash
python3 main.py
```

Расчёт с ансамблем из 50 частиц:

```bash
python3 main.py --ensemble 50
```

Расчёт с произвольными параметрами:

```bash
python3 main.py --coil-radius 0.3 --coil-current 5e5 --coil-separation 0.8 --y0 0.02 --vx0 8.2e4 --vz0 5.7e4 --t-max 2e-4 --n-steps 20000
```

Анимация в реальном времени (открывается в браузере):

```bash
python3 animate.py
python3 animate.py --vx0 2e4 --vz0 9.8e4
```

## 2. Входные параметры

Параметры задаются через аргументы командной строки.

Параметры системы:

* `--coil-radius` - радиус колец, м
* `--coil-current` - ток в кольцах, А
* `--coil-separation` - расстояние между кольцами, м

Параметры частицы:

* `--charge` - заряд, Кл
* `--mass` - масса, кг
* `--x0`, `--y0`, `--z0` - начальное положение, м
* `--vx0`, `--vy0`, `--vz0` - начальная скорость, м/с

Параметры расчёта:

* `--t-max` - время моделирования, с
* `--n-steps` - число шагов интегрирования
* `--ensemble` - число частиц для ансамблевого расчёта (только `main.py`)
* `--output` - префикс выходных файлов (только `main.py`)
* `--config` - путь к JSON-файлу с параметрами (только `main.py`)

Параметры анимации (только `animate.py`):

* `--speed` - шагов интегрирования между кадрами
* `--port` - порт веб-сервера

## 3. Ограничения входных данных

* `coil_radius` > 0
* `coil_current` != 0
* `coil_separation` > 0
* `particle_charge` != 0
* `particle_mass` > 0
* `t_max` > 0
* `n_steps` > 0
* начальное положение частицы должно быть внутри кольца
* начальное z должно быть между кольцами
* полная скорость частицы не может быть нулевой

## 4. Что считает программа

Программа:

* вычисляет магнитное поле двух колец с током
* интегрирует уравнения движения заряженной частицы алгоритмом
* определяет, удерживается частица в ловушке или вылетает
* для ансамбля частиц исследует зависимость удержания от начального угла
* сравнивает результат с теоретическим критическим углом

## 5. Структура файлов

* `config.py` - конфигурация и валидация параметров
* `field.py` - расчёт магнитного поля
* `solver.py` - интегрирование траектории (алгоритм Бориса)
* `main.py` - основная программа с визуализацией
* `demo.py` - демонстрационный скрипт
* `animate.py` - анимация в реальном времени

## 6. Физика задачи

Два кольца с током, расположенные в параллельных плоскостях, создают конфигурацию магнитной пробки. Поле на оси минимально в центре и максимально у колец. Заряженная частица с достаточно большим углом отражается от областей сильного поля и удерживается между кольцами.

Критический угол определяется зеркальным отношением:

```
sin^2(alpha_crit) = B_min / B_max
```

Частицы с начальным углом больше критического удерживаются, остальные вылетают через пробку.
